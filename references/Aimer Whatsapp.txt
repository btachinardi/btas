         - Use beforeAll/afterAll for setup/cleanup
         - Leverage cascade deletes for cleanup

         ### 3. Authentication in Tests
         - Store auth token from registration/login response
         - Pass token in Bearer auth header for subsequent requests
         - Use browser context for Playwright session management

         ### 4. Email Testing Strategy
         **Recommended Approach**:
         1. Mock email service in unit tests
         2. Use real Resend test API in integration tests
         3. Query email-events table to verify queue
         4. Use Mailslurp or similar for capturing actual emails in CI

         ### 5. Assertion Best Practices
         - Always use `{ timeout: 10000 }` for network-dependent assertions
         - Check multiple signals (URL, visible element, form state)
         - Don't assume instantaneous state changes

         ### 6. Test Data Cleanup
         - Always cleanup in afterAll/afterEach
         - Use transaction rollback for API tests
         - Delete user records that reference organizations
